## Key Principles Documented

### Quality Definition
**The trap:** Tests measuring consistency (run N matches run N-1) not correctness
**The fix:** External ground truth required (human-authored, cross-model judging, observable behavior)

### Cross-Model Independence
**Hard rule:** If Model A generated artifact, Model A cannot be sole judge
**Approaches:** Different model as judge, human ground truth with inter-rater reliability ≥ 0.70, multi-model consensus, observable behavior grounding

### Baseline Comparison Protocol
1. Identical task set
2. Baseline run (agent WITHOUT skill)
3. Treatment run (agent WITH skill)
4. Statistical comparison (p-value, effect size, confidence intervals)

**Pass criteria:**
- Treatment outperforms baseline on task completion
- No increase in false positive rate
- Statistical significance (p < 0.05, appropriate sample size)

### Statistical Validity Requirements
- Confidence intervals: Report uncertainty bounds, not just point estimates
- Sample sizes: Justify with power analysis (≥15 activation cases, ≥5 negative controls per skill; ≥10 identical tasks per condition for baseline comparison)
- Significance testing: p-values, effect sizes, multiple comparison corrections
